## Color System

Principium UI uses a sophisticated color system based on **HSL (Hue, Saturation, Lightness)** that automatically generates consistent color scales. Instead of defining individual color values, you specify hue and saturation, and the system creates a complete range of shades.

## Understanding HSL

### Hue (0-360)

The hue represents the color's position on the color wheel:

- `0` / `360` - Red
- `60` - Yellow
- `120` - Green
- `180` - Cyan
- `240` - Blue
- `300` - Magenta

### Saturation (0-100)

The saturation controls color intensity:

- `0` - Grayscale (no color)
- `50` - Muted colors
- `100` - Vivid colors

### Lightness (Generated)

Lightness values are automatically generated to create the color scale from 50 (lightest) to 950 (darkest).

## Customizing Colors

### Basic Customization

To customize colors, modify the theme configuration in your `tailwind.config.ts`:

<SyntaxHighlighter className="bg-border" language="typescript" code={`import {shadesPlugin} from '@principium/theme';

const config = {
plugins: [
shadesPlugin({
defaultTheme: 'light',
themes: {
light: {
colors: {
// Brand colors
primary: { hue: 220, saturation: 85 }, // Custom blue
secondary: { hue: 280, saturation: 70 }, // Custom purple

            // Custom brand color
            brand: { hue: 350, saturation: 80 },     // Custom red

            // Neutral adjustments
            background: { hue: 210, saturation: 5 }, // Slightly cool gray
          },
        },
      },
    }),

],
};`} />

## Adding Custom Colors

You can add completely new colors to your theme:

<SyntaxHighlighter
  className="bg-border"
  language="typescript"
  code={`colors: {
  // Existing colors
  primary: { hue: 212, saturation: 100 },
  
  // Custom colors
  accent: { hue: 290, saturation: 85 },     // Purple accent
  info: { hue: 195, saturation: 75 },       // Info blue
  neutral: { hue: 45, saturation: 5 },      // Warm neutral
  brand: { hue: 15, saturation: 90 },       // Brand orange
}`}
/>

Use custom colors in your components:

## Dark Theme Colors

Colors in dark themes often need different saturation levels:

<SyntaxHighlighter
  className="bg-border"
  language="typescript"
  code={`themes: {
  light: {
    colors: {
      primary: { hue: 212, saturation: 100 },
      background: { hue: 0, saturation: 0 },
    },
  },
  dark: {
    isDarkTheme: true,
    colors: {
      primary: { hue: 212, saturation: 85 },   // Slightly less saturated
      background: { hue: 220, saturation: 15 }, // Tinted dark background
    },
  },
}`}
/>

## Color Accessibility

### Contrast Considerations

The automatic color scale generation ensures good contrast ratios, but consider these guidelines:

- Use `*-50` to `*-200` for light backgrounds
- Use `*-700` to `*-950` for dark text
- Use `*-500` to `*-600` for medium contrast elements

## Best Practices

### 1. Maintain Consistency

Stick to your defined color palette across your application:

<SyntaxHighlighter className="bg-border"  language="tsx" code={`// ✅ Consistent with theme
<Button variant="primary">Primary</Button>
<Link className="text-primary-600">Link</Link>

// ❌ Inconsistent colors

<Button className="bg-blue-500">Button</Button>
<Link className="text-indigo-600">Link</Link>`} />

### 2. Use Semantic Names

Prefer semantic color names over specific hues:

<SyntaxHighlighter className="bg-border" language="typescript" code={`// ✅ Semantic names
colors: {
  primary: { hue: 212, saturation: 100 },
  brand: { hue: 350, saturation: 85 },
  accent: { hue: 280, saturation: 70 },
}

// ❌ Specific hue names
colors: {
blue: { hue: 212, saturation: 100 },
red: { hue: 350, saturation: 85 },
purple: { hue: 280, saturation: 70 },
}`} />

### 3. Consider Color Psychology

Choose hues that align with your brand and purpose:

- **Blue (200-240)**: Trust, stability, professionalism
- **Green (120-160)**: Growth, success, nature
- **Red (340-20)**: Energy, urgency, passion
- **Purple (260-300)**: Creativity, luxury, innovation
- **Orange (20-60)**: Enthusiasm, warmth, attention
